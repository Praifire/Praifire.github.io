DVWA之Command Injection

发表于 2018-07-23 | 分类于 DVWA

Command Injection

Command Injection，即命令注入，是指通过提交恶意构造的参数破坏命令语句结构，从而达到执行恶意命令的目的。PHP命令注入攻击漏洞是PHP应用程序中常见的脚本漏洞之一。

解决乱码问题
在文本框中输入”ip address”后结果显示为乱码，在DVWA-master\dvwa\includes目录下找到dvwaPage.inc.php文件中所有的”charset=utf-8”，修改”charset=gb2312”，即可。

low

服务器端核心代码

<?php

if( isset( $_POST[ 'Submit' ]  ) ) {
    // Get input
    $target = $_REQUEST[ 'ip' ];

没有对用户的输入进行任何的过滤或验证，而且直接使用 $_REQUEST 传递用户输入的 ip 值，这会接受用户通过 GET、POST甚至Cookies 提交的值。

// Determine OS and execute the ping command.
if( stristr( php_uname( 's' ), 'Windows NT' ) ) {
    // Windows
    $cmd = shell_exec( 'ping  ' . $target );
}
else {
    // *nix
    $cmd = shell_exec( 'ping  -c 4 ' . $target );
}

stristr() - strstr() 函数的忽略大小写版本，也就是查找字符串首次出现的位置，这里用来判断当前系统是否为 Windows 系统。

php_uname() - 返回了运行 PHP 的操作系统的相关描述。包含如下几种模式：

  • ‘a’：此为默认。包含序列 “s n r v m” 里的所有模式。
  • ‘s’：操作系统名称。例如： FreeBSD。
  • ‘n’：主机名。例如： localhost.example.com。
  • ‘r’：版本名称，例如： 5.1.2-RELEASE。
  • ‘v’：版本信息。操作系统之间有很大的不同。
  • ‘m’：机器类型。例如：i386。

shell_exec() - 通过 shell 环境执行命令，并且将完整的输出以字符串的方式返回。

命令连接符和管道:

  • | 管道符，就是将上一个命令的输出，作为下一个命令的输入
  • & 连接符，连接多条命令
  • ; 连接符，Linux下用来连接多条命令
  • && 连接符，逻辑与关系，第一个命令成功 第二个命令才会执行

  • || 连接符，逻辑或的关系，第一个命令成功 第二个命令不执行

    // Feedback for the end user

$html .= "<pre>{$cmd}</pre>";

}

?>

漏洞利用

windows

输入命令：127.0.0.1 && dir

linux

输入命令：192.168.202.137 && tree

Medium

<?php

if( isset( $_POST[ 'Submit' ]  ) ) {
// Get input
$target = $_REQUEST[ 'ip' ];

// Set blacklist
$substitutions = array(
    '&&' => '',
    ';'  => '',
);

// Remove any of the charactars in the array (blacklist).
$target = str_replace( array_keys( $substitutions ), $substitutions, $target );

// Determine OS and execute the ping command.
if( stristr( php_uname( 's' ), 'Windows NT' ) ) {
    // Windows
    $cmd = shell_exec( 'ping  ' . $target );
}
else {
    // *nix
    $cmd = shell_exec( 'ping  -c 4 ' . $target );
}

// Feedback for the end user
$html .= "<pre>{$cmd}</pre>";
}

?>

相比Low级别的代码，在 Medium 级中的代码增加使用了 str_replace 过滤用户输入中的 “&&” 和 “;” 两个连接符,即把”&&” 、”;”删除，本质上采用的是黑名单机制。str_replace把”&&”,”;”替换为空字符，因此依旧存在安全问题。

关键函数释义

str_replace — 子字符串替换

说明：

mixed str_replace ( mixed $search , mixed $replace , mixed $subject [, int &$count ] )

该函数返回一个字符串或者数组。该字符串或数组是将 subject 中全部的 search 都被 replace 替换之后的结果。

e.g.1：

<?php
$search = array('A', 'B', 'C', 'D', 'E');
$replace = array('B', 'C', 'D', 'E', 'F');
$subject = 'ABCDE';

echo str_replace($search, $replace, $subject); // output: 'FFFFFF'
?>

Because E = F according to our array,the actual output is ‘FFFFF’.

e.g.2：

<?php
// a very beatiful way to do multiple replacements is this one, using just one array
$replaceThis = Array(
'old word' => 'new word',
'was' => 'it',
'past' => 'future',
);

$originalText = "every old word was a thing of the past...";
$replacedText = str_replace(array_keys($replaceThis), $replaceThis, $originalText);
echo $replacedText;
?>

the output is :every new word it a thing of the future…

漏洞利用

  1. 因为被过滤的只有”&&”与” ;”，所以”&”不会受影响。

注意：

”&&”与” &”的区别：
A && B

先执行A，执行成功后执行B，否则不执行B.

A & B

先执行A，不管是否执行成功，都执行B.

  1. str_replace把”&&” 、”;”替换为空字符，因此可以采用以下方式绕过：

127.0.0.1&;&netstat

High

服务器端核心代码

<?php

if( isset( $_POST[ 'Submit' ]  ) ) {
// Get input
$target = trim($_REQUEST[ 'ip' ]);

// Set blacklist
$substitutions = array(
    '&'  => '',
    ';'  => '',
    '| ' => '',
    '-'  => '',
    '$'  => '',
    '('  => '',
    ')'  => '',
    '`'  => '',
    '||' => '',
);

// Remove any of the charactars in the array (blacklist).
$target = str_replace( array_keys( $substitutions ), $substitutions, $target );

// Determine OS and execute the ping command.
if( stristr( php_uname( 's' ), 'Windows NT' ) ) {
    // Windows
    $cmd = shell_exec( 'ping  ' . $target );
}
else {
    // *nix
    $cmd = shell_exec( 'ping  -c 4 ' . $target );
}

// Feedback for the end user
$html .= "<pre>{$cmd}</pre>";
}

?>

漏洞利用

”| ”（注意这里|后有一个空格）替换为空字符，于是 ”|”可以绕过。

‘| ‘ => ‘’, 这里过滤的是|加一个空格，而且这条排在了||之前，当我们输入|| whoami时，会被过滤为|whoami，||同理也可以绕过。

Impossible

服务器端核心代码

<?php

if( isset( $_POST[ 'Submit' ]  ) ) {
// Check Anti-CSRF token
checkToken( $_REQUEST[ 'user_token' ], $_SESSION[ 'session_token' ], 'index.php' );

// Get input
$target = $_REQUEST[ 'ip' ];
$target = stripslashes( $target );

// Split the IP into 4 octects
$octet = explode( ".", $target );

// Check IF each octet is an integer
if( ( is_numeric( $octet[0] ) ) && ( is_numeric( $octet[1] ) ) && ( is_numeric( $octet[2] ) ) && ( is_numeric( $octet[3] ) ) && ( sizeof( $octet ) == 4 ) ) {
    // If all 4 octets are int's put the IP back together.
    $target = $octet[0] . '.' . $octet[1] . '.' . $octet[2] . '.' . $octet[3];

    // Determine OS and execute the ping command.
    if( stristr( php_uname( 's' ), 'Windows NT' ) ) {
        // Windows
        $cmd = shell_exec( 'ping  ' . $target );
    }
    else {
        // *nix
        $cmd = shell_exec( 'ping  -c 4 ' . $target );
    }

    // Feedback for the end user
    $html .= "<pre>{$cmd}</pre>";
}
else {
    // Ops. Let the user name theres a mistake
    $html .= '<pre>ERROR: You have entered an invalid IP.</pre>';
}
}

// Generate Anti-CSRF token
generateSessionToken();

?>

使用 token 防止 CSRF 就不说了，首先使用 stripslashes 转义反斜线后的字符串，然后恐怖的来了，将输入的字符串依据’.’号分隔开，然后依次判断每段是否为整数，如果是则执行命令，如果不是则直接报错退出。
